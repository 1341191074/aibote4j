# 术语表

本章收录了 Aibote4J 框架中常用的术语及其定义。

## A

### API (Application Programming Interface)
应用程序编程接口，定义了组件之间的交互方式。

### Aibote4J
Java 跨平台 RPA 框架的名称。

### AndroidBot
Android 平台自动化机器人类。

## B

### Bot
机器人，自动化操作的执行实体。

### BotFactory
机器人工厂类，用于创建不同类型机器人的实例。

### Builder Pattern
建造者模式，一种设计模式，用于分步骤构建复杂对象。

## C

### Cache
缓存，临时存储数据以提高访问速度的机制。

### Client-Server Architecture
客户端-服务器架构，分布式系统的一种架构模式。

### Communication Strategy
通信策略，定义了客户端与服务器之间的通信方式。

### ConfigManager
配置管理器，集中管理系统配置的单例类。

### Correlation ID
关联 ID，用于追踪分布式系统中请求的唯一标识符。

## D

### Design Pattern
设计模式，软件设计中常见问题的解决方案模板。

### DTO (Data Transfer Object)
数据传输对象，用于封装数据的对象。

## H

### HID (Human Interface Device)
人机接口设备，用于连接键盘、鼠标等输入设备的标准接口。

## I

### Interface
接口，在面向对象编程中定义了一组方法契约。

## J

### Java
一种广泛使用的面向对象编程语言。

### JVM (Java Virtual Machine)
Java 虚拟机，执行 Java 字节码的虚拟机。

## M

### Maven
Java 项目的构建自动化工具。

### Microservice Architecture
微服务架构，将单一应用程序开发为一套小型服务的方法。

## N

### Netty
异步事件驱动的网络应用框架，用于快速开发高性能网络服务器。

## R

### RPA (Robotic Process Automation)
机器人流程自动化，使用软件机器人自动化业务流程的技术。

### Request Tracing
请求追踪，跟踪分布式系统中请求处理过程的技术。

### REST (Representational State Transfer)
一种软件架构风格，用于设计网络应用。

## S

### Singleton Pattern
单例模式，确保一个类只有一个实例的设计模式。

### Strategy Pattern
策略模式，定义一系列算法并将它们封装起来的设计模式。

### Template Method Pattern
模板方法模式，在父类中定义算法骨架的设计模式。

## T

### TCP/IP (Transmission Control Protocol/Internet Protocol)
互联网通信的基础协议套件。

### Thread Pool
线程池，管理一组工作线程的组件。

### Timeout
超时，操作超过指定时间未完成的状态。

## W

### WebBot
Web 平台自动化机器人类。

### WinBot
Windows 平台自动化机器人类。

## Y

### YAML (YAML Ain't Markup Language)
一种人类可读的数据序列化格式。

### YOLO (You Only Look Once)
实时物体检测系统。

## Z

### Zero Configuration
零配置，系统自动配置无需手动干预的概念。

## 符号

### API Gateway
API 网关，管理 API 请求的服务器。

### CI/CD (Continuous Integration/Continuous Deployment)
持续集成/持续部署，软件开发实践。

### JVM Options
JVM 选项，用于配置 Java 虚拟机运行参数。

### OAuth
开放授权标准，用于授权第三方访问资源。

### SSL/TLS
安全套接层/传输层安全，网络安全协议。
